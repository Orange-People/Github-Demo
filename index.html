What is GitHub?

Version Control

GitHub is built on Git, a distributed version control system that tracks changes in your code over time. Every change is recorded with a unique commit hash.

Collaboration

Teams can work together on projects using features like pull requests, code reviews, and issue tracking to maintain code quality.

Cloud Storage

Your code is stored securely in the cloud, accessible from anywhere with an internet connection. No more "it works on my machine" problems!

Essential Git Commands

Basic Commands

git init Initialize a new repository
git clone <url> Clone a repository
git status Check repository status
git add <file> Stage changes
git commit -m "message" Commit changes

Branching

git branch List branches
git checkout -b <branch> Create and switch to branch
git merge <branch> Merge branch
git branch -d <branch> Delete branch

Remote Operations

git push origin <branch> Push to remote
git pull origin <branch> Pull from remote
git fetch Fetch remote changes
git remote -v List remotes

Best Practices

Meaningful Commit Messages

Write clear, descriptive commit messages that explain what and why, not how.

Good: "Add user authentication with JWT tokens"
Bad: "Fix stuff"

Feature Branches

Always create a new branch for each feature or bug fix. Never work directly on main.

git checkout -b feature/user-dashboard

Code Reviews

Always have someone else review your code before merging. Use pull requests for all changes.

Create PR → Request review → Address feedback → Merge

README Documentation

Keep your README.md updated with setup instructions, project description, and usage examples.

Include: Description, Installation, Usage, Contributing, License
